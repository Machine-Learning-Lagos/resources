# Contributing to Machine Learning Lagos - Resources Repository

Thank you for your interest in contributing to the **Machine Learning Lagos** resources repository! This repository is a place where we gather valuable resources to support our community in learning, collaborating, and building exciting AI projects. We’re excited to have you on board and can’t wait to see your contributions!

## How You Can Contribute

There are several ways you can contribute to this repository:

1. **Add Study Group Materials**: Contribute new or updated materials for our study groups. These might include guides, reading materials, or tutorials.
2. **Share General Resources**: Add resources that provide foundational knowledge or insights into various AI fields.
3. **Projects**: Submit a project idea or contribute project templates for other community members to build on.
4. **Create Tutorials**: Share tutorials on essential tools and frameworks, especially those related to Google AI products.
5. **Provide Event Materials**: Upload slides, recordings, or notes from past events.

## Contribution Guidelines

To maintain consistency and ensure high-quality contributions, please follow these guidelines:

### 1. Fork the Repository

Fork this repository to your own GitHub account. This will allow you to make changes without affecting the original repository.

### 2. Create a Branch

Create a branch in your forked repository for your contribution. Use descriptive branch names (e.g., `add-intro-to-ml-tutorial` or `update-general-resources`).

### 3. Add Your Contribution

- Place your files in the appropriate folder:
  - **/study-groups**: Resources for study groups on specific topics.
  - **/general-resources**: Foundational or industry-related AI resources.
  - **/projects**: Project templates.
  - **/tutorials**: Step-by-step guides or tutorials on tools/frameworks.
  - **/events**: Resources from past events or upcoming event materials.
- For new topics, create a Markdown file (`.md`) and use descriptive names (e.g., `intro-to-ml.md` or `ai-in-healthcare.md`).
- Make sure to check for typos, format text and writing style should be consistent, and keep file sizes manageable.

### 4. Write a Clear Commit Message

Write a clear, concise commit message describing your changes. Example: `Added tutorial for intro to machine learning` or `Updated project templates for new projects`.

### 5. Submit a Pull Request (PR)

Once you're happy with your changes, submit a pull request to the main repository. In your pull request:

- Provide a brief summary of what you’ve added or changed.
- Include any relevant details, such as why this resource is valuable to the community.
- Tag any related issues if applicable.

Our team will review your pull request and provide feedback if needed. Once approved, your contribution will be merged!

## Code of Conduct

We want Machine Learning Lagos to be a welcoming and collaborative environment. Please:

- Be respectful and constructive in all interactions.
- Avoid submitting duplicate content—check the repository to ensure your contribution is unique or adds new value.
- Ensure all contributions adhere to our community guidelines and are suitable for a diverse audience.

## Contact Us

If you have any questions or need assistance, feel free to reach out! You can open an issue in this repository or connect with us through [Twitter](https://x.com/mlinlagos) and [LinkedIn](https://www.linkedin.com/company/machine-learning-lagos/).

Thank you for contributing to Machine Learning Lagos! Together, we’re building a valuable resource for our community.
